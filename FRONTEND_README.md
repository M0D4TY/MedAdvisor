# 医疗顾问系统 - 简单前端使用指南

## 概述

本项目提供了两个前端界面：

1. **完整前端** (`app/web/templates/index.html`) - 通过FastAPI服务器渲染的Jinja2模板
2. **简易前端** (`simple_frontend.html`) - 独立的HTML文件，可直接在浏览器中打开

## 简易前端使用说明

### 1. 文件位置
- `simple_frontend.html` - 项目根目录下的独立HTML文件

### 2. 启动步骤

#### 步骤1：启动后端服务器
```bash
# 方法1：使用启动脚本（推荐）
cd d:\VsCode\CC
python run_server.py

# 方法2：手动启动
cd d:\VsCode\CC
set PYTHONPATH=%CD%
python -m uvicorn app.api.app:create_app --host 127.0.0.1 --port 8000
```

#### 步骤2：打开前端界面
1. 在浏览器中打开 `simple_frontend.html` 文件
   - 直接双击文件，或
   - 在浏览器地址栏输入：`file:///d:/VsCode/CC/simple_frontend.html`

2. 等待状态栏显示"已连接"
   - 绿色圆点表示连接成功
   - 灰色圆点表示未连接

#### 步骤3：开始使用
- **发送消息**：在输入框中输入问题，点击"发送"或按Enter键
- **快速提问**：点击上方的快速问题按钮
- **查看响应**：对话会显示在聊天区域中

### 3. 功能特点

#### 界面布局
- **顶部标题栏**：显示应用名称和简介
- **快速问题区**：常用问题的快捷按钮
- **聊天消息区**：显示用户和助手的对话
- **输入区域**：文本输入框和发送按钮
- **状态栏**：显示连接状态、API地址和响应时间

#### 交互功能
- **实时聊天**：与医疗AI助手对话
- **快速提问**：一键发送预设问题
- **自动滚动**：新消息自动滚动到可视区域
- **打字指示**：等待响应时显示打字动画
- **响应时间**：显示每次请求的响应时间
- **连接状态**：实时监控服务器连接状态

#### 技术特性
- **纯前端技术**：HTML5 + CSS3 + JavaScript，无外部依赖
- **响应式设计**：适配桌面和移动设备
- **现代化UI**：渐变背景、圆角设计、平滑动画
- **错误处理**：网络错误时的友好提示
- **会话管理**：自动维护对话session

### 4. API接口

前端连接的后端API：
- `POST http://localhost:8000/api/chat` - 聊天接口
- `GET http://localhost:8000/health` - 健康检查
- `GET http://localhost:8000/api/version` - 版本信息

### 6. 故障排除

#### 问题1：无法连接到服务器
**症状**：状态栏显示"未连接"，灰色圆点
**解决方案**：
1. 确保后端服务器正在运行（端口8000）
2. 检查控制台是否有错误信息
3. 在浏览器中直接访问 `http://localhost:8000/health` 测试连接

#### 问题2：发送消息无响应
**症状**：点击发送后无反应
**解决方案**：
1. 检查浏览器开发者工具的控制台（F12）
2. 查看网络请求是否被发送
3. 检查服务器日志是否有错误

#### 问题3：跨域问题（CORS）
**症状**：浏览器控制台显示CORS错误
**解决方案**：
后端已配置CORS中间件，允许所有来源访问：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 允许所有来源
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

#### 问题4：页面显示异常
**症状**：样式错乱或布局异常
**解决方案**：
1. 刷新页面（Ctrl+F5强制刷新）
2. 检查浏览器是否支持现代CSS特性
3. 尝试其他浏览器（Chrome、Firefox、Edge）

### 7. 开发说明

#### 自定义修改
要修改前端界面，可编辑 `simple_frontend.html` 文件：

1. **修改样式**：编辑 `<style>` 标签内的CSS代码
2. **修改布局**：编辑HTML结构
3. **修改交互**：编辑 `<script>` 标签内的JavaScript代码
4. **修改API地址**：修改 `const API_BASE = 'http://localhost:8000';`

#### 扩展功能
可添加的功能建议：
- 对话历史记录
- 消息复制功能
- 主题切换（深色/浅色模式）
- 语音输入支持
- 文件上传（症状图片等）

### 8. 与完整前端的区别

| 特性 | 简易前端 | 完整前端 |
|------|----------|----------|
| **部署方式** | 独立HTML文件 | Jinja2模板，通过FastAPI渲染 |
| **依赖** | 无外部依赖 | 依赖htmx库 |
| **功能** | 基本聊天功能 | 更丰富的功能和UI组件 |
| **样式** | 内联CSS | 更复杂的CSS样式 |
| **交互** | 原生JavaScript | 使用htmx进行部分交互 |
| **适用场景** | 快速测试、简单展示 | 生产环境、完整功能 |

### 9. 浏览器兼容性

- **推荐浏览器**：Chrome 90+、Firefox 88+、Edge 90+、Safari 14+
- **所需特性**：ES6、Flexbox、CSS Grid、Fetch API
- **移动端**：iOS Safari、Android Chrome

### 10. 安全注意事项

1. **本地使用**：前端设计为本地使用，API地址固定为localhost
2. **生产部署**：如需公网访问，需配置HTTPS和适当的安全措施
3. **数据安全**：聊天内容通过HTTP传输，敏感信息请勿在对话中分享
